## üìù FIELD GUIDELINES

### Step 1: Agent Army Status
- Check all 11 agents' current load
- Status: ‚úÖ Ready / ‚ö†Ô∏è High Load / ‚ö†Ô∏è Overloaded
- Current Load: Percentage from AGENT_UTILIZATION_TRACKER.md
- Available For: Task types the agent can handle based on load

### Step 2: Dates & Events Today
- Current Session: Phase, Day, Progress percentage
- Important Events:
  - Completed tasks (check PERFORMANCE_LOG.md)
  - Upcoming deadlines (check USERS_PREFERENCES.md)
  - Special events (check MEMORY.md)

### Step 3: Yesterday's Performance
- Session: Date range
- Tasks Completed:
  - List from PERFORMANCE_LOG.md or DAILY_ACTIVITIES_LOG.md
- Issues Encountered:
  - From PERFORMANCE_LOG.md
- Improvements:
  - Optimizations implemented

### Step 4: Cost Summary (7 Days)
- Daily Cost: From PERFORMANCE_LOG.md
- Daily Average: Calculate over 7 days
- Free Model Usage: Percentage from PERFORMANCE_LOG.md
- Cost Per Task: Estimated from total tasks / total cost

### Step 5: Today's Priorities (Ranked)
- Review PERFORMANCE_LOG.md for pending tasks
- Check WEEK3_PLAN.md for Week 3 priorities
- Rank by High/Medium/Low
- Assign appropriate agent (based on task type and load)

**High Priority (Today):** Tasks must complete today  
**Medium Priority (This Week):** Tasks required this week  
**Low Priority (Backlog):** Nice-to-have tasks

### Step 6: Proactive Suggestions
**From HEARTBEAT.md:**
- Review HEARTBEAT.md for pending checks
- Any system improvements needed
- Optimization opportunities

**From OPTIMIZATION_QUICK_START.md:**
- Review quick wins
- Optimization tasks
- Best practices

**From agent analysis (atlas subagent):**
- Underutilized agents (titan, max, vision)
- Coordination improvements
- Testing opportunities
